## When to use this skill

- When writing git commit messages
- When creating pull request titles and descriptions
- When staging changes with `git add`
- When reviewing `git diff` before committing
- When formatting PR descriptions with Summary, Changes, Testing sections
- When using active voice verbs (Add, Update, Fix, Remove, Refactor)
- When following single-line commit message format (no prefixes)
- When linking PRs to tickets or related issues
- When describing changes for code reviewers
